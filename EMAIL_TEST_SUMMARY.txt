HOW TO USE ON RENDER (3 SIMPLE STEPS)
================================================================================

STEP 1: Add Environment Variables to Render
--------------------------------------------
1. Go to Render Dashboard: https://dashboard.render.com/
2. Select your PandaPickle web service
3. Go to: Environment → Environment Variables
4. Click "Add Environment Variable" and add these THREE variables:

   Variable 1:
   Key:   BREVO_SMTP_USER
   Value: [Your Brevo SMTP user]

   Variable 2:
   Key:   BREVO_SMTP_PASS
   Value: [Your Brevo SMTP API key]

   Variable 3:
   Key:   BREVO_FROM_EMAIL
   Value: [Your verified sender email]

5. Click "Save Changes"
6. Wait for Render to redeploy (usually 2-3 minutes)

⚠️ NOTE: Get actual values from your .env file or Brevo dashboard

STEP 2: Open the Email Tester
------------------------------
1. Open browser
2. Go to: https://pandapickle.onrender.com/test_email.php?key=pandapickle2026
3. You'll see a professional testing interface

STEP 3: Send Test Email
------------------------
1. Enter your email address (e.g., naningb16@gmail.com)
2. Click "Send Test Email"
3. Check your inbox (and spam folder!)
4. You should receive a professional HTML email within 1-2 minutes

================================================================================
TROUBLESHOOTING
================================================================================

IF YOU SEE "Configuration Error":
----------------------------------
❌ Problem: Environment variables not set in Render
✅ Solution:
   1. Go to Render Dashboard
   2. Check Environment Variables section
   3. Verify all three variables are present
   4. Check for typos in variable names (must be exact!)
   5. Wait for Render to finish redeploying

IF YOU SEE "Email Failed to Send":
-----------------------------------
❌ Problem: SMTP connection or authentication failed
✅ Solutions:
   1. Check Render logs for detailed error message
   2. Verify SMTP password is correct (no spaces)
   3. Check Brevo dashboard: https://app.brevo.com/
   4. Verify sender email is verified in Brevo
   5. Check if daily limit (300 emails) is reached

IF EMAIL NOT RECEIVED:
----------------------
❌ Problem: Email sent but not in inbox
✅ Solutions:
   1. Check spam/junk folder (most common!)
   2. Wait 2-5 minutes (sometimes delayed)
   3. Check Brevo dashboard → Statistics for delivery status
   4. Verify email address is spelled correctly
   5. Try different email address (Gmail, Yahoo, etc.)
