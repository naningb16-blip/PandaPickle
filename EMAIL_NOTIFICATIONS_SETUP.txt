EMAIL CONFIGURATION:
--------------------

**FOR LOCAL XAMPP:**
PHP's mail() function won't work on localhost by default.
You have two options:

Option 1: Use a test email service (recommended for testing)
- Install a local SMTP server (e.g., Papercut, MailHog)
- Or use a service like Mailtrap.io for testing

Option 2: Configure XAMPP to send real emails
- Edit php.ini:
  [mail function]
  SMTP = smtp.gmail.com
  smtp_port = 587
  sendmail_from = your-email@gmail.com
  auth_username = your-email@gmail.com
  auth_password = your-app-password

**FOR RENDER (PRODUCTION):**
Render supports PHP's mail() function out of the box.
Emails will be sent from the server's sendmail.

IMPORTANT NOTES:
----------------
⚠️ Email delivery is not guaranteed with PHP mail()
⚠️ Emails may go to spam folders
⚠️ For production, consider using a service like:
   - SendGrid (free tier: 100 emails/day)
   - Mailgun (free tier: 5,000 emails/month)
   - AWS SES (pay as you go)

TROUBLESHOOTING:
----------------
❌ Emails not sending on localhost
   → This is normal. Configure SMTP or use Mailtrap for testing

❌ Emails going to spam
   → Add SPF and DKIM records to your domain
   → Use a reputable email service (SendGrid, Mailgun)

❌ Function sendBookingConfirmationEmail() not found
   → Make sure includes/email.php is included at top of file

❌ Email sent but customer didn't receive it
   → Check spam folder
   → Verify customer's email address is correct in database
   → Check server mail logs
