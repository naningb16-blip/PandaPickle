## 📋 PRE-DEPLOYMENT CHECKLIST

### Step 1: Verify Files Are Updated
[ ] Check reservations.php has duplicate prevention code

### Step 2: Backup Current Database
[ ] Export current database via phpMyAdmin
[ ] Save backup file with date: pandapickle_backup_2026-06-20.sql
[ ] Store backup in safe location

**Command**:
```bash
mysqldump -u root -p pandapickle > pandapickle_backup_$(date +%Y%m%d).sql
```

### Step 3: Test Locally First
[ ] Run test script: http://localhost/PandaPickle/test_security.php
[ ] Verify all tests pass (especially password hashing)
[ ] Test duplicate reservation manually

[ ] Test normal operations still work

## 🗄️ DATABASE MIGRATION

### Step 4: Apply Security Indexes

**Choose ONE method:**

### Method A: MySQL Command Line (Recommended)
[ ] Open terminal/command prompt
[ ] Navigate to project folder:
    cd C:\xampp\htdocs\PandaPickle
[ ] Run migration:
    mysql -u root -p pandapickle < database/migration_add_security_indexes.sql
[ ] Verify success (no error messages)

### Method B: phpMyAdmin (Easier)
[ ] Open phpMyAdmin: http://localhost/phpmyadmin
[ ] Select database: pandapickle
[ ] Click "SQL" tab
[ ] Open file: database/migration_add_security_indexes.sql
[ ] Copy ALL content
[ ] Paste in SQL window
[ ] Click "Go" button
[ ] Verify: "9 rows affected" or similar success message

### Method C: XAMPP Shell
[ ] Open XAMPP Control Panel
[ ] Click "Shell" button
[ ] Run commands:
    mysql -u root -p
    use pandapickle;
    source C:/xampp/htdocs/PandaPickle/database/migration_add_security_indexes.sql;
    exit;

### Step 5: Verify Indexes Were Created

## 🧪 LOCAL TESTING

### Step 6: Test All Security Features

### Test A: Password Hashing
[ ] Register new test user: test_security@example.com
[ ] Check database: SELECT password FROM users WHERE email = 'test_security@example.com';
[ ] Password should start with: $2y$
[ ] Try to login with correct password: ✅ Should work
[ ] Try to login with wrong password: ❌ Should fail
[ ] Delete test user

### Test B: Duplicate Email Prevention
[ ] Register user: duplicate_test@example.com
[ ] Try to register same email again
[ ] Should show error: "Email is already registered"
[ ] Check database: Only 1 user with this email exists
[ ] Delete test user

### Test C: Duplicate Reservation Prevention
[ ] Login as customer
[ ] Create reservation: Court 1, Tomorrow, 10:00 AM, 2 hours
[ ] Note reservation ID in database
[ ] Refresh page and submit EXACT same reservation
[ ] Should show error: "You already have a reservation..."
[ ] Check database: Only 1 reservation created (verify ID)
[ ] Delete test reservation

### Test D: Duplicate Registration Prevention
[ ] Login as customer
[ ] Register for open play session (random match)
[ ] Note registration ID in database
[ ] Go back and try to register for same session again
[ ] Should show error: "You are already registered for this session"
[ ] Check database: Only 1 registration created (verify ID)
[ ] Delete test registration

### Test E: Transaction Safety
[ ] Create reservation with payment
[ ] Check database: Both reservation AND payment record exist
[ ] Try to create reservation but cancel midway (close browser)
[ ] Check database: No orphaned records
[ ] All records are complete or don't exist

### Test F: Normal Operations Still Work
[ ] Create court reservation → Should work ✅
[ ] Cancel reservation → Should work ✅
[ ] Register for open play → Should work ✅
[ ] Register with friends (4 players) → Should work ✅
[ ] Admin approve registration → Should work ✅
[ ] Generate matches → Should work ✅
[ ] Mark match as finished → Should work ✅
