ADMIN DASHBOARD REDIRECT FIX
==============================================

Date: June 21, 2026
Status: ✅ COMPLETE

PROBLEM:
--------
Admins were accessing the same dashboard as regular users (dashboard.php), 
which showed customer-specific information like their personal reservations 
and open play registrations. This was confusing since admins should see 
system-wide statistics and management tools.

SOLUTION:
---------
Implemented automatic redirection so admins always see the admin dashboard 
(admin/index.php) and customers always see the customer dashboard (dashboard.php).

CHANGES MADE:
-------------

1. ✅ login.php
   - After successful login, checks user role
   - Admins redirected to: admin/index.php
   - Customers redirected to: dashboard.php

2. ✅ dashboard.php
   - Added check at the top of the file
   - If logged-in user is admin, immediately redirect to admin/index.php
   - Prevents admins from manually accessing customer dashboard

3. ✅ includes/auth.php
   - Updated requireAdmin() redirect path
   - Non-admin users trying to access admin pages redirected to ../dashboard.php
   - Fixed relative path issue (was missing ../)

BEHAVIOR NOW:
-------------
✅ Admin logs in → Goes to admin/index.php (admin dashboard with statistics)
✅ Customer logs in → Goes to dashboard.php (customer dashboard with their bookings)
✅ Admin tries to visit dashboard.php → Automatically redirected to admin/index.php
✅ Customer tries to visit admin/* → Denied and redirected to dashboard.php with error
