PAYMENT FLOW:
-------------

USER SIDE:
1. User logs in → dashboard.php
2. Sees reservations/registrations with "💳 Pay Now" button
3. Clicks Pay Now → redirected to payment-info.php
4. Views payment details (GCash/Bank info)
5. Makes payment via GCash or Bank Transfer
6. Uploads receipt image
7. Sees "Receipt uploaded successfully!" message
8. Payment status: unpaid → pending_verification

ADMIN SIDE:
1. Admin goes to admin/payments.php
2. Sees all payments with uploaded receipts
3. Clicks receipt thumbnail to view full image
4. Clicks "Verify Paid" or "Reject"
5. Payment status: pending_verification → paid/rejected
6. User can see updated status on dashboard

DATABASE STRUCTURE:
-------------------

payments table:
- id (primary key)
- reservation_id (foreign key)
- registration_id (foreign key)
- payment_type ('reservation' or 'open_play')
- proof_image (path to uploaded receipt)
- amount (decimal)
- payment_status ('unpaid', 'pending_verification', 'paid', 'rejected')
- verified_by (admin user_id)
- verified_at (timestamp)

payment_settings table:
- id (primary key)
- setting_key (gcash_number, bank_name, etc.)
- setting_value (actual value)
- setting_label (display label)
- is_active (boolean)
- display_order (integer)

PAYMENT STATUSES:
-----------------
1. unpaid - Initial status, no payment made
2. pending_verification - Receipt uploaded, awaiting admin review
3. paid - Admin verified and approved payment
4. rejected - Admin rejected payment, user must re-upload
