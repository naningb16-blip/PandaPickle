========================================
POSTGRESQL QUOTE SYNTAX FIXES
========================================

ISSUE:
------
After fixing SQL function compatibility, encountered errors with PostgreSQL quote syntax:

Error Example:
  SQLSTATE[42703]: Undefined column: 7 ERROR: column "active" does not exist
  LINE 1: SELECT * FROM courts WHERE status = "active" ORDER BY court_...

ROOT CAUSE:
-----------
In PostgreSQL:
- Single quotes ('') are for STRING VALUES: WHERE status = 'active'
- Double quotes ("") are for IDENTIFIERS (table/column names): "my_table"."my_column"

MySQL is more lenient and allows double quotes for both, but PostgreSQL is strict.

FIXES APPLIED:
--------------

1. **String Literals in SQL Queries** - Changed double quotes to single quotes:
   
   ❌ BEFORE: WHERE status = "active"
   ✅ AFTER:  WHERE status = 'active'
   
   ❌ BEFORE: AND status IN ("pending", "approved")
   ✅ AFTER:  AND status IN ('pending', 'approved')
   
   ❌ BEFORE: VALUES (?, "reservation", ?, "paid")
   ✅ AFTER:  VALUES (?, 'reservation', ?, 'paid')

2. **IF() Function** - Replaced with CASE WHEN (PostgreSQL syntax):
   
   ❌ BEFORE: IF(r.user_id IS NULL, "walk-in", "online") as booking_type
   ✅ AFTER:  CASE WHEN r.user_id IS NULL THEN 'walk-in' ELSE 'online' END as booking_type

3. **RAND() Function** - Replaced with RANDOM() (PostgreSQL syntax):
   
   ❌ BEFORE: ORDER BY reg.match_preference DESC, RAND()
   ✅ AFTER:  ORDER BY reg.match_preference DESC, RANDOM()
