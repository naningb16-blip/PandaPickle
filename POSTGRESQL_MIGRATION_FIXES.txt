POSTGRESQL MIGRATION - SQL FUNCTION FIXES
========================================

ISSUE:
------
After deploying to Render with PostgreSQL, the application was throwing errors because it was using MySQL-specific SQL functions that don't exist in PostgreSQL.

Error Example:
  SQLSTATE[42883]: Undefined function: 7 ERROR: function curdate() does not exist

MYSQL vs POSTGRESQL SQL FUNCTIONS:
-----------------------------------

1. CURDATE() → CURRENT_DATE
   MySQL: WHERE date_column >= CURDATE()
   PostgreSQL: WHERE date_column >= CURRENT_DATE

2. DATE_SUB(NOW(), INTERVAL X MINUTE/DAY) → NOW() - INTERVAL 'X minutes/days'
   MySQL: WHERE created_at > DATE_SUB(NOW(), INTERVAL 5 MINUTE)
   PostgreSQL: WHERE created_at > NOW() - INTERVAL '5 minutes'

3. NOW() - Works in both (no change needed)
