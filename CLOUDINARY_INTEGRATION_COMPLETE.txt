========================================
HOW IT WORKS
========================================

USER UPLOADS RECEIPT:
1. User clicks "Upload Receipt" button
2. File sent to payment-info.php
3. File validated (type, size)
4. uploadToCloudinary() called
5. File uploaded via Cloudinary API
6. Cloudinary returns secure URL
7. URL saved to database (proof_image column)
8. Success message shown to user

ADMIN VIEWS RECEIPT:
1. Admin goes to Verify Payments page
2. Query gets proof_image from database
3. Image tag uses full Cloudinary URL
4. Image loads from Cloudinary CDN
5. Click to view full size

DATABASE STORAGE:
- Old: proof_image = 'uploads/receipts/receipt_123.jpg' ❌
- New: proof_image = 'https://res.cloudinary.com/deiy19kv4/image/upload/v1234/pandapickle/receipts/receipt_123' ✅

TROUBLESHOOTING
========================================

Fix: Check internet connection, verify credentials

Issue: Image not displaying in admin panel

Issue: cURL error
Fix: Ensure cURL is enabled in PHP (should be by default in XAMPP)
