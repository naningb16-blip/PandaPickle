PAYMENT RECEIPT UPLOAD FEATURE - COMPLETE
========================================

OVERVIEW:
---------
Users can now upload payment receipts after clicking "Pay Now" on their reservations or open play registrations.
Admins can then verify these receipts and approve/reject payments.

IMPLEMENTATION DETAILS:
-----------------------

1. USER FLOW:
   - User clicks "💳 Pay Now" button on dashboard
   - Redirected to payment-info.php showing payment details (GCash, Bank info)
   - User makes payment via GCash or Bank Transfer
   - User uploads receipt image (JPG, PNG, GIF up to 5MB)
   - Receipt stored in uploads/receipts/ directory
   - Payment status changes to "pending_verification"
   - User sees confirmation message

2. ADMIN VERIFICATION FLOW:
   - Admin visits admin/payments.php
   - Sees all payments with status badges
   - Can view uploaded receipt thumbnails (click to enlarge)
   - Can "Verify Paid" or "Reject" payments
   - Verified payments marked with admin ID and timestamp

3. FILES MODIFIED:
   ✅ payment-info.php - Added receipt upload form and handler
   ✅ admin/payments.php - Already has receipt display and verification (existing)
   ✅ dashboard.php - Already has "Pay Now" buttons (existing)
   ✅ database/schema.sql - proof_image column already exists
   ✅ assets/css/style.css - proof-thumb style already exists

4. FILES CREATED:
   ✅ uploads/receipts/.gitkeep - Directory for receipt uploads
   ✅ uploads/.gitignore - Protect uploaded files from git
   ✅ database/migration_add_payment_method.sql - Optional payment method tracking

5. DATABASE STRUCTURE:
   payments table already has:
   - proof_image VARCHAR(255) - Stores path to uploaded receipt
   - payment_status - Has 'pending_verification' status
   - verified_by - Admin ID who verified
   - verified_at - Timestamp of verification

6. PAYMENT STATUSES:
   - unpaid: No payment made yet
   - pending_verification: Receipt uploaded, waiting for admin review
   - paid: Admin verified payment
   - rejected: Admin rejected payment (user must re-upload)

7. SECURITY FEATURES:
   - File type validation (only JPG, PNG, GIF)
   - File size limit (5MB max)
   - Unique filename generation (prevents overwrites)
   - User can only upload for their own bookings
   - Receipts stored outside public access (uploads/ folder)

8. MIGRATIONS TO RUN:
   ✅ database/migration_add_payment_settings.sql (REQUIRED - creates payment settings table)
   ⚠️ database/migration_add_payment_method.sql (OPTIONAL - adds payment_method and admin_notes columns)
